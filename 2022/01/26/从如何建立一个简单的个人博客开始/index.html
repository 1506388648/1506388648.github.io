从如何建立一个简单的个人博客开始

发表于 2022-01-26 更新于 2022-01-29 分类于 创建个人博客
本文字数： 1.3k 阅读时长 ≈ 1 分钟
借助Hexo以及Github搭建个人博客（环境：win10）

软件准备

安装Git

Git的下载地址：https://git-scm.com/
Git的安装和使用：https://www.cnblogs.com/ximiaomiao/p/7140456.html
使用SSH秘钥链接Github：http://www.xuanfengge.com/using-ssh-key-link-github-photo-tour.html

安装Nodejs

网站

官方网站：https://nodejs.org/en/ （我选择的是目前最新的长期最新支持版本16.13.2）
以往版本：https://nodejs.org/zh-cn/download/releases/

安装过程

按顺序依次安装即可
在 GitBrash中使用命令 node -v ,npm -v（npm：Node package manager，在安装Node的时候已经一起安装了）查看版本号

输入下面命令安装 hexo

在Git brash中输入hexo -v检查是否安装成功

然后在任意位置建一个存放博客的文件夹
右键那个文件夹，选择Git brash
在Git brash中输入hexo init blog初始化博客文件夹
输入ls -l查看获取的文件
输入hexo s(下面出现很多的warning，是版本高，不满足一些依赖导致的，但是并不影响hexo
程序的使用，解决的方法可以Node降级到12，但是后面需要在文章上设置文章字数和阅读时长的功能时需要用到hexo-symbols-count-time插件，会出现需要依赖的版本提示，如12.22.0 14.17.0或者大于16.0.0 )

使用浏览器网址输入http://localhost:4000/会看到Hellow Word的一个静态页面，即运行成功。
但是这只是在我们本地端口查看，hexo s被打断了就不能查看了，所以我们通常使用hexo s来进行本地预览，达到效果后在部署到网站上。

将网站部署到GitHub上

在自己的GitHub上新建一个仓库。

仓库的命名应该是自己的昵称+.github.io，例如我的昵称是JayChen125那么我仓库的命名是JayChen125.github.io

在博客目录下右键选择GitBrash，安装一个hexo部署的插件

npm install --save hexo-deployer-git

安装成功后，在博客文件夹目录下找到_config.yml 的配置文件，建议使用Visual Studio Code打开，当然记事本也可以。找到如下代码：

type : 一个名字，建议写成git方便识别
repo : repository仓库地址 将刚刚新建的github仓库地址复制到这里就可以了
branch: 分支这里设置管理员 master
